**Some Links, Tips & Thoughts for the day**

**Android Studio -**
  I have really been enjoying working with Android Studio Version 3 the last couple of weeks as it has been 4 years since
  I previously used the software and then it was a new release and Eclipse was still ruling the roost. I have found it to be
  as responsive as Visual Studio has been on Windows which is high praise as the Visual Studio experience is for most
  developers among the best they have.
  
  Among my dislikes of other IDE has been the difficulty in setting up a working Github repo for a project but Android Studio
  is very easy, the process is as follows
  
  - Login to github.com and create a new repository.
  
  - Initialise the repository with README unticked
  
  - Open Android Studio and create a new project
  
  - Setup the options that are needed for the project
  
  - Once the project is open, on the top menu select VCS > Import into Version Control > Create Git Repository
  
  - Open terminal available on the platform and navigate to the root of the project
  
  - Add the remote repo using git remote add origin <repo_name.git> and press enter
  
  - In Android Studio, right click your projects root directory and select Git > Add
  
  - Next, right click the project name again and select Git > Commit Directory
  
  - Type a Commit Message and select Commit
  
  - A Code Analysis warning may appear, depending on the error repair or click Commit
  
  - Right click the project name, select Git > Repository > Push
 
 There are keyboard shortcuts available but it is better to get this process down first and then use the shortcuts to
 improve workflow. This is a very quick process, as quick as using the cli which I like but I am slowly getting used to
 a using the IDE inbuilt tools
  


**Knowing When To Change Course -**
  I have recently been trying out Java Spring, version 5 for some sample projects as you can run React, Angular etc alongside
  the framework without any issues. I have to say that it is not for me, I found some of the framework to be much more
  unwieldy than it should be. This is not a criticism of the framework at all, this framework covers a multitude of arenas
  and quite successfully but I felt as though it definitely is something that belongs in the enterprise.
  
  This gave rise to a situation a lot of developers and other IT people seem to be going through which is keeping on 
  learning something because they feel that they should. I did struggle with Spring and some of it was down to not being
  fully engaged with the technology so I did something for the first time, I put the tech down and focused on something
  else which I both enjoy and will be of benefit to me. I hope that this does not happen too much as I have a limited
  cognitive capability and I enjoy many different technology areas.
